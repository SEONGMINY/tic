일정 리마인더 - tic 사용성 개선

### 1. 요구사항

- 일간 캘린더 화면(DayView.swift)의 주간 스트립 섹션에서, 타임라인 섹션과 마찬가지로 Slide in / Slide out 애니메이션을 적용한다.
- 일간 캘린더 화면(DayView.swift)에서 비어있는 타임라인을 long press 하면 일간 생성 화면이 나타난다. 이때 첨부한 이미지처럼 임시 타임라인 블록이 생성되며, 일정 생성 Bottom Sheet가 나타나난다. 일정 생성 완료를 마치지 않았다면 임시 생성된 블록은 제거된다.
- 일간 캘린더 화면(DayView.swift)에서 생성된 타임라인블록 클릭 시. 일정 수정 화면이 나타난다. 타임라인블록을 long press 시 context menu 와 오른쪽 상단 꼭지점과 왼쪽 하단 꼭지점이 활성화 된다. 꼭짓점 드래그를 통해 타임라인 블록의 범위를 손쉽게 수정할 수 있으며, 15분 단위로 움직일 수 있다. 꼭짓점이 활성화 된다는건 타임라인 블록을 업데이트 할 수 있는 모드가 활성화 된다는 것이며, 드래그 앤 드롭 기능을 통해 타임라인블록을 손쉽게 옮기는 기능도 제공한다. context menu 의 경우, [삭제 / 복제 ] 만 제공한다.
- 일간 캘린더 화면(DayView.swift)에서 타임라인블록이 생성되어있는 경우, 일간 생성 화면을 보일 수 없기 때문에, 타임라인의 왼쪽 시간을 나타내는 UI 에서도 long press 기능을 제공한다. 타임라인 섹션과 시간을 나타내는 UI 의 간격을 조금 더 준다.
- 일정 생성 시간화면에서, 캘린더 드롭다운을 통해 캘린더 / 미리 알림을 선택할 수 있는데 첨부한 이미지처럼 toggle 을 통해 캘린더를 생성할지 미리 알림을 생성할지 선택할 수 있다. 기본적으로 첨부한 이미지의 UI 처럼 싱크를 맞춘다. 기본적으로 border radius 가 많이 들어간 스타일을 제공한다.
- 년간 캘린더 화면(YearView.swift)에서 "오늘" 을 "올해" 로 텍스트 변경한다. 월간 캘린더 화면(MonthView.swift)에서도 년간 캘린더 화면(YearView.swift)에 있는 "올해" 기능을 제공한다. 텍스트는 "이번달" 이며, 위치는 동일하게 왼쪽 하단에 위치한다. 해당 버튼 클릭 시 올해의 이번달로 스크롤한다.
- live-activity 와 Dynamic Island 화면을 전면적으로 개편한다. 오늘의 일정을 하나의 선과 progress UI 를 통해서 타임라인을 표시한다. 첨부한 이미지처럼 이미 지나간 시간은 실선으로 표기하고 앞으로 남은 일정은 점선으로 표기한다.

### 2. 구현 디테일

- SwiftUI 를 사용하여 네이티브 앱을 만든다.
- 데이터는 로컬에만 저장한다.
- 최대한 구현이 쉬운 기술을 사용한다.
- 그 외 기술은 적절히 너가 추천해서 사용해야한다.

### 3. 제약 조건

- 첨부한 이미지의 UI 와 기능을 최대한 싱크를 맞추어야한다.
- `docs/*` 의 문서들을 최대한 참고하여 현재까지 결정한 사항들과 의사결정 문맥을 잃지 말아야한다.

### 4. 완료 조건

- 일간 캘린더 화면(DayView.swift)에서 여러 zindex 때문에 제스처나 기능이 동작하지 않는 경우가 있는데, 기능들을 모두 제공해야한다.
- 핵심 작동 흐름이 정상 동작한다.

지금부터 위 요구사항을 개선하기 위한 설계를 시작할 것이다. 첫 단계는 구현 가능성 검증이다. 이 서비스가 실제로 구현가능할지 기술적으로 검증해야한다. 너는 나의 공동 창업자고, 우리의 목표는 빠르게 스타트업을 만들어 다음 Y Combinator batch 에 합격하는거다. 그걸 위해서 빠르게 MVP 를 출시하고 시장의 반응을 보면서, 여러 가설을 테스트해야한다. 속도가 생명이다. 그러면서도 안정적인 서비스를 만들어야한다. 이 둘간의 트레이드오프를 명심해라. 그럼 지금부터 논의를 시작하자. 너가 주도적으로 구현 가능성을 점검하고, 조금이라도 모호하거나 구체화 해야할 것 이 있다면 나와 논의하자.
